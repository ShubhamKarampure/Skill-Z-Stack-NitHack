## üí° The Problem vs. The Solution

### The "Trust Gap"
* **Resume Fraud:** 34% of employers hire blindly due to fake credentials.
* **Slow Verification:** Background checks take weeks and cost ~$100/check.
* **Siloed Data:** LinkedIn, Universities, and MOOCs don't talk to each other.
* **Centralized Risk:** If a university server goes down, your degree disappears.

### The Skill-Z Approach
Skill-Z bridges this gap by issuing **tamper-proof NFT credentials** verified instantly via blockchain. We utilize **Zero-Knowledge Proofs (ZKP)** to allow students to prove their skills/age without revealing sensitive personal data, and **AI-driven visualization** to map their growth.

---

## üöÄ Key Features

* **üõ°Ô∏è Privacy-First Verification:** Uses **ZK-SNARKs (Circom)** to prove age or credential ownership without revealing the actual data.
* **üåå 3D Skill Constellations:** An interactive, AI-powered (Google Gemini) 3D graph (Three.js) that visualizes a student's skills and recommends new "Ghost Nodes" to learn.
* **‚õΩ Gas-Aware Minting:** Smart contracts monitor network Gwei to batch-mint credentials cost-effectively.
* **üèõÔ∏è DAO Governance:** Institutes are voted in based on Reputation Scores; bad actors are slashed via community voting.
* **üîó Interoperable:** Metadata stored on IPFS, fully compatible with standard HR tools via REST APIs.

## ‚ö° Local Development Setup

Follow these steps strictly to set up the environment with **Ganache** and **Truffle**.

### 1\. Prerequisites

  * Node.js (v18+)
  * Truffle (`npm install -g truffle`)
  * Ganache CLI (`npm install -g ganache`)
  * MongoDB (Running locally or Atlas URI)

### 2\. Blockchain Environment (Ganache)

We use a specific Ganache configuration for deterministic testing.

**Open a terminal and run:**

```bash
ganache -a 50 -e 1000 -p 7545 -d -b 2
```

  * `-a 50`: Creates 50 accounts
  * `-e 1000`: 1000 ETH per account
  * `-p 7545`: Port
  * `-d`: Deterministic (Same addresses every time)
  * `-b 2`: Block time in seconds

### 3\. Wallet Configuration

Create a file named `available-wallets.txt` in a `constants` folder (in frontend) to store the accounts generated by Ganache for easy access during testing.

```text
Available Accounts
==================
(0) 0x90F8bf6A479f320ead074411a4B0e7944Ea8c9C1 (1000 ETH)
... [Copy from Ganache Terminal]

Private Keys
==================
(0) 0x4f3edf98... [Copy from Ganache Terminal]
```

### 4\. Smart Contract Deployment

Navigate to the blockchain folder and deploy the contracts.

```bash
cd blockchain
npm install

# Compile contracts
truffle compile

# Migrate to local Ganache network
truffle migrate --reset --network development
```

**‚ö†Ô∏è CRITICAL STEP:**
After migration, Truffle generates a file `development_addresses.json` (or similar output in build). You must copy the contract addresses and paste them into `server/config/development_addresses.json` (or your specific config path) so the backend knows where to point.

### 5\. Backend Setup

```bash
cd server
npm install

# Configure .env (see below)
# Start the server
npm run start
```

### 6\. Frontend Setup

```bash
cd frontend
npm install

# Configure .env.local (see below)
# Start the Next.js app
npm run dev
```

## üì° API Endpoints Overview

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| **POST** | `/auth/register` | Register Student, Institute, or Employer |
| **POST** | `/credentials/issue` | Mint NFT Credential (Institute only) |
| **GET** | `/verify/verify/:id` | Verify ZK Proof for a credential |
| **POST** | `/issuers/accredit` | DAO Voting for new Institutes |
| **GET** | `/enrollments/institutes` | List accredited institutions |
